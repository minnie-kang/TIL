# SA 문서 토대로 데이터베이스 설계


1. User 테이블
- 기본적인 사용자 정보관리
2. SocialAccount 테이블
- 소셜 로그인 정보 관리
3. UserCourse 테이블
- 사용자가 수강한 코스 정보 관리    
4. Course 테이블
- 코스 정보 관리
5. Lesson 테이블
- 강의 정보 관리
6. Quiz 테이블
- 퀴즈 정보 관리
7. PracticeExercise 테이블
- 실습 문제 정보 관리   
8. Post 테이블
- 게시글 정보 관리
9. AI_Interaction 테이블
- AI 응답 정보 관리
10. LearningRoadmap 테이블
- 학습 로드맵 정보 관리
11. RoadmapStep 테이블
- 로드맵 단계 정보 관리
12. UserRoadmapProgress 테이블
- 사용자 로드맵 진행 정보 관리
13. Performance 테이블
- 성취 정보 관리
14. UserPerformance 테이블
- 사용자 성취 정보 관리

```mermaid
erDiagram
    User {
        bigint id PK
        varchar email UNIQUE
        varchar password NULL
        varchar google_id NULL
        varchar username
        timestamp created_at
        timestamp updated_at
    }
    SocialAccount {
        int id PK
        bigint user_id FK > User.id
        string provider
        string social_id
        timestamp created_at
        timestamp updated_at
    }
    UserCourse {
        int id PK
        bigint user_id FK > User.id
        bigint course_id FK > Course.id
        enum status
        int progress_percentage
        timestamp enrolled_at
        timestamp completed_at
        timestamp last_accessed_at
        timestamp created_at
        timestamp updated_at
    }
    Course {
        bigint id PK
        varchar title
        text description
        tinyint difficulty_level
        timestamp created_at
        timestamp updated_at
    }
    Lesson {
        int id PK
        bigint course_id FK > Course.id
        string title
        text content
        int order
        timestamp created_at
        timestamp updated_at
    }
    Quiz {
        int id PK
        int lesson_id FK > Lesson.id
        string question
        text answer_options
        string correct_answer
        timestamp created_at
        timestamp updated_at
    }
    PracticeExercise {
        int id PK
        int lesson_id FK > Lesson.id
        string title
        text description
        text initial_code
        text solution_code
        text test_cases
        timestamp created_at
        timestamp updated_at
    }
    Post {
        int id PK
        bigint user_id FK > User.id
        string title
        text content
        timestamp created_at
        timestamp updated_at
    }
    AI_Interaction {
        int id PK
        bigint user_id FK > User.id
        text prompt
        text response
        timestamp created_at
        timestamp updated_at
    }
    LearningRoadmap {
        int id PK
        string title
        text description
        timestamp created_at
        timestamp updated_at
    }
    RoadmapStep {
        int id PK
        int roadmap_id FK > LearningRoadmap.id
        string title
        text content
        int step_order
        timestamp created_at
        timestamp updated_at
    }
    UserRoadmapProgress {
        int id PK
        bigint user_id FK > User.id
        int roadmap_step_id FK > RoadmapStep.id
        enum status
        timestamp started_at
        timestamp completed_at
        timestamp created_at
        timestamp updated_at
    }
    Performance {
        int id PK
        string name
        text description
        int target_percentage
        timestamp created_at
        timestamp updated_at
    }
    UserPerformance {
        int id PK
        bigint user_id FK > User.id
        int performance_id FK > Performance.id
        float achievement_rate
        timestamp achieved_at
        timestamp created_at
        timestamp updated_at
    }
